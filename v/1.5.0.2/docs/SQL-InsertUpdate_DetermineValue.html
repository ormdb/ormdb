Some columns need to be inserted/updated with their default/initial values. Specify SetInitials parameter as true for setting columns with their initial-value. This parameter is false by default.
Rules are shown below for determining the value for updating or inserting processes.
Value Rule Result
Null/Nothing
{
1 - Column Default Value (CDV) :
If the value is "Null or Nothing" and especially CDV was specified on column definition; then CDV is accepted.
2 - Column Initial Value (CIV) :
If the value and CDV are "Null or Nothing" and SetInitial was specified as true on insert or update processes; then CIV is accepted.

Column Initial Values
{
ColumnString : String.Empty
ColumnNumeric : 0 (Zero)
ColumnBoolean : false
ColumnBinary : Array.Empty
ColumnTime : TimeSpan.Zero
ColumnDate : Date.MinValue.Date
ColumnDateTime : Date.MinValue
}
3 - DbNull :
If the value and CDV and CIV are "Null or Nothing"; then DbNull is accepted.
}
 CDV, CIV or DbNull
DbNull If NULL value will be used especially on insert or update processes; then DbNull value has to be specified. DbNull
Other All values except "Null/Nothing" and "DbNull". Value is accepted as is. As Is
