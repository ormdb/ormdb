To retrieve data, create an instance of a SQL.IQueryable object, and then records can be read by calling ExecuteReader. The ExecuteReader provides an unbuffered stream of data that allows procedural logic to efficiently process results from a data source sequentially. The ExecuteReader is a good choice when you're retrieving large amounts of data because the data is not cached in memory. Definition Namespace : Odb These methods are come as an extension methods. So you have to include or import to use them.

Examples
Following example iterates the records for two columns. 

		// 1. to execute IQueryable.
		var sql = SQL.X.Select(h.Note, h.DocDate).From(h).Where(h.Year == 2017);
		var dr = sql.ExecuteReader(cn);

		// 2. to execute TSQL string.
		var sql2 = "SELECT h.Note, h.DocDate FROM TableHeader AS h WHERE h.Year == 2017";
		var dr = sql2.ExecuteReader(cn);

		// 3. to execute DbCommand.
		var cmd = cn.CreateCommand(
						"SELECT h.Note, h.DocDate FROM TableHeader AS h WHERE h.Year=@param1",
						new { param1 = 2017});
		var dr = cn.ExecuteReader(cmd);


		// looping records.
		foreach (var r in dr) {
			// read by column-name.
			Debug.WriteLine("{0}, {1}", r("Note").toString(), 
										r("DocDate").toString());

			// read by column.
			Debug.WriteLine("{0}, {1}", r(h.Note).toString(), 
										r(h.DocDate).toString());

			// read by column-index.
			Debug.WriteLine("{0}, {1}", r(0).toString(),
										r(1).toString());
		}
