Columns (type of Odb.Table.Column) are also defined by supplying necessary type information in the attribute. If a column-type is often used in many of your tables, so you can use a column as a reference. The column-types as shown below, can be used in the table definitions.


You can use connection.Provider.ToDbTypeName to get the Database Equivalent statement of the column. Example shown below, returns different results for each provider type. 

        //-- [column1] is ColumnNumeric(4, 0)
        var result = cn.Provider.ToDbTypeName(column1);
        
        //-- MsSql      = numeric(4, 0)
        //-- MySql      = DECIMAL(4, 0)
        //-- Oracle     = NUMBER(4, 0)
        //-- PostgreSql = NUMERIC(4, 0)
	
        '//-- [column1] is ColumnNumeric(4, 0)
        Dim result = cn.Provider.ToDbTypeName(column1)
        
        '//-- MsSql      = numeric(4, 0)
        '//-- MySql      = DECIMAL(4, 0)
        '//-- Oracle     = NUMBER(4, 0)
        '//-- PostgreSql = NUMERIC(4, 0)



Column Types Namespace : Odb.Table
All column types has the properties as shown below.
Property Name Description
c - ColumnName Optional. It is the name of the column.
d - Description Optional. It is the description of the column.
n - Nullable Optional. This constraint enforces a column to accept or NOT accept NULL values.
v - DefaultValue Optional. It is the default value of the column.
ColumnString (Length) is used for .NET System.String data type. Variable-size string data. Length is the size in bytes, NOT in char. Max value of Length can be different according to databases. 

			[ColumnString(10)]public Column col_char;
	
			<ColumnString(10)>Public col_char AS Column
ColumnNumeric (Precision, Scale) is used for any .NET numeric types such as int[16/32/64], sigle, double, decimal etc. Precision is an integer that represents the total number of digits allowed in this column. The scale, also an integer value, represents the number of decimal places to right of the decimal point. 

			[ColumnNumeric(15, 2)]public Column col_num;
	
			<ColumnNumeric(15, 2)>Public col_num AS Column
ColumnBoolean is used for .NET System.Boolean data type. 

			[ColumnBoolean()]public Column col_bool;
	
			<ColumnBoolean()>Public col_bool AS Column
ColumnBinary is used for .NET System.Byte[] data type. 

			[ColumnBinary()]public Column col_bin;
	
			<ColumnBinary()>Public col_bin AS Column
ColumnTime is used for .NET System.TimeSpan data type. This type of column stores only TIME value of System.TimeSpan as "24hh:mm:dd". 

			[ColumnTime()]public Column col_tim;
	
			<ColumnTime()>Public col_tim AS Column
ColumnDate is used for .NET System.Date data type. This type of column stores only DATE value of System.Date as "yyyy-MM-dd". 

			[ColumnDate()]public Column col_dat;
	
			<ColumnDate()>Public col_dat AS Column
ColumnDateTime is used for .NET System.Date data type. This type of column stores DATE and TIME value of System.Date as "yyyy-MM-dd 24hh-mm-ss". 

			[ColumnDateTime()]public Column col_dattim;
	
			<ColumnDateTime()>Public col_dattim AS Column
ColumnRef is used for columns used repeatedly. Name, Description and Nullable properties can be redefiend. 

			[ColumnRef( reference )]public Column col_ref;
	
			<ColumnRef( reference )>Public col_ref AS Column

ColumnInt (Length)   Deprecated There is no INTEGER data type for all databases because an INTEGER is a number with no decimal precision in ANSI standards.
SubType Accept As Byte Size
Int8 / tinyint ColumnNumeric(3, 0) 3
Int16 / smallint ColumnNumeric(5, 0) 5
Int32 / int ColumnNumeric(10, 0) 9
Int64 / bigint ColumnNumeric(19, 0) 9
These types are not supported for definition only. But you can allways use these types in database.
